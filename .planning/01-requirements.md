# CafeHub — Requirements & Use Cases

## 📖 사용자 시나리오 (User Stories)

### 시나리오 1️⃣: 직장인의 미팅 준비 — "점심시간 회의 자리 확보"

**배경**
- 영준(28, 직장인)은 오후 2시에 팀장과 카페에서 1시간 미팅 예정
- 지금은 오후 1시 30분
- 강남역 근처에서 조용하면서도 분위기 좋은 카페를 찾아야 함

**과정**
```
1️⃣ 앱 열기
   → 지도 화면 표시, 현재 위치 자동 감지

2️⃣ "강남역 조용한 카페" 검색
   → 검색어 입력 또는 지도에서 반경 선택

3️⃣ 필터링
   - 분위기: "모임/회의용" 선택
   - 시설: "아메리카노 있음", "충전 가능" 체크
   - 별점: 4.5점 이상만 표시

4️⃣ 카페 선택
   - 지도에 표시된 3개 카페 중 리뷰 확인
   - 사진으로 실제 분위기 체크
   - "YC 카페" 선택 (평점 4.7, 리뷰 127개)

5️⃣ 예약 진행
   - 날짜: 오늘
   - 시간: 오후 2시
   - 인원: 2명
   - 소요 시간: 1시간
   - 확인: "오후 2시~3시, 창가 자리 예약 가능"

6️⃣ 결제
   - 선 결제 또는 현장 결제 선택 (선택)
   - 신용카드/카카오페이 결제
   - 예약 확인번호 발급

7️⃣ 출발 및 입장
   - 예약 확인 화면에서 카페 주소 클릭
   - 지도 앱으로 자동 이동
   - 카페 도착 → 예약 확인번호 제시
   - 즉시 자리 제공 ✅
```

**결과**
- ✅ 시간 낭비 제로 (도착 후 자리 대기 없음)
- ✅ 팀장과 편하게 미팅 진행
- ✅ 이후 리뷰 작성 가능

**시스템이 제공해야 할 것**
- [x] 위치 기반 검색
- [x] 필터링 (분위기, 시설)
- [x] 카페 정보 조회 (사진, 리뷰, 평점)
- [x] 예약 시스템 (날짜/시간/인원)
- [x] 결제 통합
- [x] 예약 확인

### 시나리오 2️⃣: 학생의 장시간 공부 — "4시간 집중 공부하기"

**배경**
- 지은(24, 학생)은 내일 시험이 있음
- 오전 10시~오후 2시 (4시간) 집중해서 공부하고 싶음
- 집에서는 산만해서 카페에서 공부하는 것을 선호
- 신림역 근처 조용한 카페를 찾음

**과정**
```
1️⃣ 앱 열기

2️⃣ "신림역 스터디 카페" 검색
   → 위치: 신림역 반경 500m
   → 키워드: "스터디", "조용"

3️⃣ 필터링
   - 분위기: "조용함" (필수)
   - 시설: "WiFi", "충전", "화장실 깨끗함"
   - 가격대: 아메리카노 4,000원 이하

4️⃣ 카페 선택 과정
   - 지도에 5개 카페 표시
   - 리뷰에서 "4시간 공부했어요", "정말 조용해요" 찾기
   - 사진으로 실제 책상 간격 확인
   - "공부방 카페" 선택 (조용함 평점 4.9)

5️⃣ 예약
   - 날짜: 내일
   - 시간: 10:00
   - 기간: 4시간 (10:00 ~ 14:00)
   - 인원: 1명
   - 특수 요청: "창가 자리, 조용한 곳"

6️⃣ 결제 (선택)
   - 아메리카노 + 치즈케이크 미리 주문
   - 또는 현장에서 주문

7️⃣ 공부
   - 카페에서 4시간 집중력 있게 공부
   - WiFi와 충전소 활용
   - 필요시 화장실 이용 ✅

8️⃣ 리뷰 작성
   - "4시간 공부 완벽했어요! 정말 조용합니다"
   - 사진: 책상 사진
   - 별점: 5점
```

**결과**
- ✅ 예약으로 자리 확보 불안감 해소
- ✅ 4시간 집중 환경 제공
- ✅ 마음 편하게 공부 완료

**시스템이 제공해야 할 것**
- [x] 상세한 카페 정보 (시설, 분위기, 사진)
- [x] 신뢰할 수 있는 리뷰 (실제 사용자 후기)
- [x] 장시간 예약 지원
- [x] 특수 요청사항 입력 (특정 자리, 메모)
- [x] 예약 확인 화면에서 카페 정보 언제든 확인

### 시나리오 3️⃣: 커플의 데이트 준비 — "분위기 좋은 카페에서 2시간"

**배경**
- 민준(26)과 수연(26)이 토요일 오후 데이트 계획
- 분위기 좋은 카페에서 2시간 시간 보내고 싶음
- 홍대 지역에서 "감성적이고 예쁜" 카페 찾기

**과정**
```
1️⃣ 앱 열기 (커플이 함께 또는 한 명이 먼저)

2️⃣ "홍대 감성 카페" 또는 "데이트 카페" 검색
   → 위치: 홍대역 반경 1km

3️⃣ 필터링
   - 분위기: "감성적", "분위기 좋음"
   - 가격대: 아메리카노 5,500원~
   - 시설: "인생샷 명소", "분위기"

4️⃣ 카페 선택
   - 지도에 10개 카페 표시
   - 각 카페의 사진 슬라이더 넘기며 보기
   - "비슷한 분위기" 추천 확인
   - 남자 리뷰와 여자 리뷰 모두 확인
   - "플라워 카페" 선택 (분위기 4.8, 여성 평점 4.9)

5️⃣ 예약
   - 날짜: 토요일
   - 시간: 오후 4시
   - 기간: 2시간
   - 인원: 2명
   - 특수 요청: "창가 자리, 감성 좋은 곳" (있으면)

6️⃣ 결제 (선택)
   - 미리 결제 또는 현장 결제
   - 메뉴: 아메리카노 2개 + 케이크 세트

7️⃣ 데이트
   - 카페 방문
   - 편안한 분위기에서 시간 보내기
   - 사진 촬영 (감성 좋은 배경)
   - 대화 나누기 ✅

8️⃣ 리뷰 작성
   - "데이트에 완벽했어요! 분위기 정말 좋습니다"
   - 사진: 카페 내부, 음료 사진
   - 별점: 5점
```

**결과**
- ✅ 분위기 좋은 장소에서 편하게 시간 보냄
- ✅ 대기 없이 즉시 입장
- ✅ 기억에 남는 데이트 완성

**시스템이 제공해야 할 것**
- [x] 감성적인 사진 전시 (여러 장, 슬라이더)
- [x] 상세 리뷰 (남녀 구분, 데이트 후기 태그)
- [x] 분위기 평점 (일반 평점과 분리)
- [x] 비슷한 카페 추천
- [x] 예약 확인 후에도 카페 정보 언제든 확인

## 📋 기능 요구사항 (Functional Requirements)

### MoSCoW 분류

#### 🔴 MUST (반드시 해야 함) — 6개

```
1. 회원가입 & 로그인
   - 이메일, Google, Apple 로그인 지원
   - 프로필 관리 (이름, 사진, 선호도)

2. 카페 검색 & 필터링
   - 위치 기반 검색 (Google Maps)
   - 필터: 분위기, 시설, 가격대, 별점
   - 지도에 마커 표시

3. 카페 상세 정보 조회
   - 사진, 주소, 영업시간, 연락처
   - 평점 & 리뷰
   - 시설 정보 (WiFi, 충전, 화장실 등)

4. 자리 예약 시스템
   - 날짜, 시간, 인원 선택
   - 예약 상황 표시 (가능/만석)
   - 예약 확인 & 취소

5. 결제 통합
   - Stripe 연동
   - 신용카드, 카카오페이, 토스 지원
   - 영수증 발급

6. 카페 리뷰 & 평가
   - 별점 (1~5점)
   - 텍스트 리뷰
   - 사진 업로드 (최대 3장)
   - 리뷰 조회
```

#### 🟡 SHOULD (있으면 좋음) — 4개

```
1. 찜한 카페 저장
   - 즐겨찾기 기능
   - 저장한 카페 목록 조회

2. 카페 방문 기록
   - 예약 내역 조회
   - 방문한 카페 통계

3. 친구와 예약 공유
   - 예약 번호 공유
   - 함께 가기 기능

4. 카페 추천 기능
   - 비슷한 분위기 추천
   - 방문자들이 많이 함께 방문한 카페
```

#### 🟢 COULD (나중에) — 3개

```
1. AI 기반 추천
   - 사용자 취향에 맞는 카페 추천
   - "당신을 위한 카페" 섹션

2. 카페 사업자 대시보드
   - 예약 관리
   - 통계 조회

3. 이벤트 & 쿠폰
   - 시즈널 이벤트 공지
   - 할인 쿠폰 제공
```

#### ⚪ WON'T (이번 프로젝트에서는 안 함)

```
1. 음성 주문
2. 결제 시 할인/캐시백
3. 매장 가입 기능
4. 다국어 지원
```

## 🏛️ 비기능 요구사항 (Non-Functional Requirements)

### 성능 (Performance)

| 항목 | 목표 | 측정 기준 |
|------|------|---------|
| 앱 로딩 속도 | < 3초 | 첫 화면 표시까지 |
| 지도 로딩 | < 2초 | 카페 마커 표시까지 |
| 검색 응답 | < 1초 | 검색 결과 표시까지 |
| 예약 완료 | < 2초 | 결제 후 확인까지 |

### 보안 (Security)

```
- HTTPS 통신만 사용
- 비밀번호 암호화 (bcrypt, 6자 이상)
- API 인증 (JWT 토큰)
- 결제 정보 PCI DSS 준수
- 민감 정보 마스킹 (번호, 이메일)
```

### 신뢰성 (Reliability)

```
- 앱 크래시: 0건 (중대 버그)
- API 가용성: 99.5% 이상
- 데이터 손실: 0건
- 백업: 일일 자동 백업
```

### 사용성 (Usability)

```
- 회원가입: 3분 이내
- 예약: 클릭 5번 이내
- 앱 크기: < 100MB
- 지원 언어: 한국어 (영어는 추후)
```

### 확장성 (Scalability)

```
- 동시 사용자: 100명 이상 지원
- 예약 처리: 초당 10건 이상
- 데이터 저장: 100MB 이상
```

Users
├─ userId (PK)
├─ email
├─ name

├─ cafeId (FK)

├─ bookingTime
├─ duration
├─ people
