#### 02. 의사결정 로그 (ADR) 작성 ⏳

**상태**: ⏳ **10주차 예정**

**설명**:
- 기술 선택에 대한 의사결정 이유를 ADR 형식으로 문서화
- 최소 3개 이상 작성

**계획 내용**:

**ADR-0001: Mobile Framework 선택 — Flutter**
```
배경:
- iOS/Android 동시 개발 필요
- 7주 제한 시간

선택지:
1️⃣ Flutter (선택)
   ✅ Dart 언어, 빠른 개발, 핫 리로드
   ❌ Dart 학습 필요

2️⃣ React Native
   ✅ JavaScript 사용 가능
   ❌ 성능 이슈, 커뮤니티 복잡도

3️⃣ 네이티브 (Swift + Kotlin)
   ✅ 최고 성능
   ❌ 개발 시간 2배 이상

최종 결정: Flutter
이유: "7주 내 iOS/Android 동시 배포" 달성 가능
```

**ADR-0002: Backend 선택 — Firebase**
```
배경:
- 서버 개발 비용 최소화
- 실시간 업데이트 필요

선택지:
1️⃣ Firebase (선택)
   ✅ 무료 Tier, 실시간 Firestore, 인증 통합
   ❌ 벤더 락인, 고비용 시 이전 어려움

2️⃣ AWS
   ✅ 유연성 높음
   ❌ 설정 복잡, Lambda 비용 높을 수 있음

3️⃣ 자체 서버 구축 (Node.js)
   ✅ 완전 제어
   ❌ 호스팅 비용, 서버 관리 시간

최종 결정: Firebase
이유: "개발 속도 최우선" — 인증, 데이터베이스, 저장소 통합
```

**ADR-0003: Architecture Pattern — Layered + MVVM**
```
배경:
- 한 명 개발자, 6주 기한
- 테스트 가능한 구조 필요

선택지:
1️⃣ Layered + MVVM (선택)
   ✅ 학습 곡선 낮음, 테스트 용이
   ❌ 보일러플레이트 많음

2️⃣ Clean Architecture
   ✅ 장기 유지보수 우수
   ❌ 학습 시간 높음, 복잡함

3️⃣ MVC
   ✅ 간단함
   ❌ 테스트 어려움, 확장성 낮음

최종 결정: Layered + MVVM
이유: "6주 내 배포" & "테스트 가능성" 모두 달성
```

**예상 일정**: 11월 15~20일

### Tier 3: 혁신 가산점 (각 5%)

#### 05. Google Maps 고급 기능 — "카페 클러스터링" ⏳

**상태**: ⏳ **12-13주차 예정**

**설명**:
- 지도에 많은 마커가 있을 때 자동으로 클러스터링
- 사용자 경험 개선 (성능 + 시각성)

**구현 내용**:
```
기능:
- 줌 레벨에 따라 마커 자동 그룹화
- 클러스터 수 표시 (예: "23")
- 클러스터 클릭 시 자동 줌 인
- 클러스터 분해 애니메이션

기술:
- Google Maps Clustering 라이브러리
- 또는 custom 클러스터링 알고리즘 (Firebase GeoHash 활용)

기대 효과:
- 지도 로딩 시간 50% 단축
- 마커 표시 성능 3배 향상
```

**예상 일정**: 12월 3~5일

#### 06. "찜한 카페" 기능 + 로컬 데이터베이스 ⏳

**상태**: ⏳ **12-13주차 예정**

**설명**:
- 사용자가 선호하는 카페를 저장
- 오프라인 상태에서도 접근 가능
- 로컬 SQLite/Hive 데이터베이스 활용

**구현 내용**:
```
기능:
1. 찜하기 (하트 아이콘)
   - 카페 상세 화면에서 ❤️ 버튼
   - Firestore + 로컬 DB에 저장
   
2. 찜한 카페 목록
   - 별도 탭 ("❤️ 찜한 카페")
   - 오프라인에서도 볼 수 있음
   - 로컬에 캐싱된 사진, 리뷰 표시

3. 찜한 카페 통계
   - "가장 많이 찜한 분위기"
   - "평가 높은 순서대로"
   - "새로 추가된 카페"

기술:
- Hive 또는 SQLite (로컬 데이터베이스)
- sync_local_with_firestore 로직
- 오프라인 감지 (connectivity 패키지)
```

**예상 일정**: 12월 6~10일

#### 07. 푸시 알림 (Local Notification) ⏳

**상태**: ⏳ **13주차 예정**

**설명**:
- 예약 30분 전 알림
- 예약 당일 아침 리마인더

**구현 내용**:
```
알림 시나리오:

   내용: "14:00 예약 | 주소 복사하기"
   
2. 예약 당일 아침 (로컬)
   제목: "오늘의 카페 예약을 확인하세요"
   내용: "YC 카페 14:00 | 일정에 추가"

3. 새 리뷰 (Firebase Cloud Messaging은 나중에)
   - 찜한 카페에 새 리뷰 작성되면 알림

기술:
- flutter_local_notifications 패키지
- work_manager (백그라운드 스케줄링)
- scheduled notifications
```
