## 1. 기능 요구사항 (Functional Requirements)

### FR-1: 회원 관리

#### FR-1.1 회원가입
```
요구사항: 사용자가 앱을 통해 새로운 계정을 만들 수 있어야 한다.
입력항목:
  - 아이디 (중복 검사 포함)
  - 비밀번호 (6자 이상)
  - 이름
  - 이메일
  - 전화번호 (선택)
출력: 가입 완료 메시지, 자동 로그인
제약: 아이디, 이메일 중복 불가
```

#### FR-1.2 로그인
```
요구사항: 사용자가 아이디/비밀번호로 로그인할 수 있어야 한다.
입력항목:
  - 아이디
  - 비밀번호
출력: JWT 토큰 발급, 홈 화면 이동
제약: 최대 5회 실패 시 30분 잠금
```

#### FR-1.3 자동 로그인
```
요구사항: 저장된 토큰이 유효하면 자동 로그인되어야 한다.
조건:
  - 토큰 유효 시간: 24시간
  - 리프레시 토큰: 30일
동작: 앱 실행 시 자동으로 홈 화면 이동
```

#### FR-1.4 프로필 관리
```
요구사항: 사용자가 자신의 정보를 수정할 수 있어야 한다.
수정 가능 항목:
  - 이름
  - 이메일
  - 비밀번호
  - 전화번호
```

### FR-2: 카페 검색 및 조회

#### FR-2.1 카페 검색
```
요구사항: 사용자가 다양한 조건으로 카페를 검색할 수 있어야 한다.
검색 방법:
  1. 키워드 검색 (카페명)
  2. 지역 검색 (구/동)
  3. 카테고리 필터 (분위기, 시설)
반환: 매칭되는 카페 리스트
성능: 검색 결과 1초 이내 표시
```

#### FR-2.2 카페 상세 조회
```
요구사항: 카페의 상세 정보를 조회할 수 있어야 한다.
표시 정보:
  - 카페 명칭
  - 위치 (지도 표시)
  - 영업 시간
  - 시설 정보 (와이파이, 콘센트, 주차 등)
  - 분위기 평가 (조용함/활기참)
  - 평균 별점
  - 최근 리뷰 3개
  - 예약 가능 여부
```

---

### FR-3: 예약 시스템

#### FR-3.1 예약 생성
```
요구사항: 사용자가 카페를 예약할 수 있어야 한다.
예약 항목:
  - 카페 선택
  - 방문 날짜 (오늘~30일)
  - 방문 시간 (10:00~19:00, 30분 단위)
  - 인원 수 (1~10명)
검증:
  - 해당 시간대 자리 확인
  - 최대 인원 제한 확인
결과: 예약 확정, 예약 번호 부여
```

#### FR-3.2 예약 조회
```
요구사항: 사용자가 자신의 예약 내역을 조회할 수 있어야 한다.
표시 정보:
  - 카페 명칭
  - 예약 일시
  - 인원 수
  - 예약 상태
  - 예약 번호
필터: 전체/예약 대기/예약 확정/방문 완료
```

#### FR-3.3 예약 취소
```
요구사항: 사용자가 예약을 취소할 수 있어야 한다.
취소 조건:
  - 방문 예정 시간 24시간 이전만 가능
  - 방문 완료된 예약은 취소 불가
결과: 예약 취소 완료, 상태 변경
```

### FR-4: 리뷰 시스템

#### FR-4.1 리뷰 작성
```
요구사항: 사용자가 방문한 카페에 대한 리뷰를 작성할 수 있어야 한다.
입력항목:
  - 카페 선택
  - 별점 (1~5점)
  - 리뷰 텍스트 (최대 300자)
  - 사진 첨부 (선택)
검증:
  - 방문 완료된 예약만 리뷰 작성 가능
  - 중복 리뷰 불가
```

#### FR-4.2 리뷰 조회
```
요구사항: 모든 사용자가 작성된 리뷰를 조회할 수 있어야 한다.
표시 정보:
  - 작성자 이름 (별칭)
  - 별점
  - 리뷰 텍스트
  - 작성 날짜
  - 유용성 투표 (도움됨/도움 안됨)
정렬: 최신순/별점순/유용순
```

#### FR-4.3 리뷰 삭제
```
요구사항: 자신이 작성한 리뷰는 삭제할 수 있어야 한다.
제약:
  - 본인 리뷰만 삭제 가능
  - 삭제 후 복구 불가
```

## 2. 비기능 요구사항 (Non-Functional Requirements)

### NFR-1: 성능 (Performance)

```
NFR-1.1 응답 시간
  - 검색 결과: 1초 이내
  - 예약 생성: 2초 이내
  - 리뷰 로드: 1.5초 이내

NFR-1.2 동시 사용자
  - 최대 동시 접속: 10,000명
  - 예약 초당 처리: 100건

NFR-1.3 데이터 동기화
  - 실시간 동기화 지연: 500ms 이내
  - 오프라인 모드: 지원
```

### NFR-2: 보안 (Security)

```
NFR-2.1 인증
  - JWT 토큰 기반 인증
  - 토큰 유효 시간: 24시간
  - HTTPS 통신 필수

NFR-2.2 암호화
  - 비밀번호: BCrypt 해싱
  - 전송 데이터: TLS 1.2+
  - 민감 정보: 암호화 저장

NFR-2.3 권한 관리
  - 사용자는 자신의 데이터만 접근 가능
  - 관리자 기능 분리
```

### NFR-3: 가용성 (Availability)

```
NFR-3.1 시스템 가용성
  - 목표 가용성: 99.5% 이상
  - 최대 장애 시간: 월 4시간 이내

NFR-3.2 백업
  - 일일 백업
  - 백업 보관 기간: 30일
```

### NFR-4: 확장성 (Scalability)

```
NFR-4.1 사용자 확장
  - 초기: 1,000명
  - 6개월: 10,000명
  - 12개월: 50,000명 목표

NFR-4.2 데이터 확장
  - 카페 정보: 5,000개
  - 예약 데이터: 월 10만건
```

### NFR-5: 호환성 (Compatibility)

```
NFR-5.1 플랫폼
  - Android: 버전 10 이상
  - iOS: 버전 12 이상

NFR-5.2 화면 크기
  - 최소: 4.7 인치 (스마트폰)
  - 최대: 7.0 인치 (태블릿)
```

## 4. 사용자 시나리오 (User Scenarios)

### 시나리오 1: 카페 검색 및 예약
```
1. 사용자가 앱 실행 → 로그인 화면
2. 로그인 또는 자동 로그인
3. 홈 화면에서 "검색" 탭 선택
4. "강남역 근처" 입력, "조용한 분위기" 필터 선택
5. 매칭되는 카페 5개 표시
6. 카페 A 선택 → 상세 정보 확인
7. "예약" 버튼 클릭
8. 날짜(내일), 시간(14:00), 인원(2명) 선택
9. "예약 확정" → 예약 번호 부여
10. 확인 알림 표시
```

### 시나리오 2: 방문 후 리뷰 작성
```
1. "예약 조회" 탭 → 예약 내역 확인
2. 방문 완료된 예약 선택
3. "리뷰 작성" 버튼 클릭
4. 별점 4점 선택
5. "커피가 맛있고 직원이 친절했어요" 입력
6. "등록" 버튼 클릭
7. 리뷰 작성 완료 확인
```
