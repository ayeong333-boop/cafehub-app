7️⃣ 기술 스택 선택 근거

Flutter

iOS/Android 동시 개발 가능, 7주 일정에 최적화, 핫 리로드로 빠른 개발

Firebase

인증/DB 무료, 실시간 데이터 동기화, 배포 간단, 확장성 우수

Google Maps API

정확한 위치 기반 서비스, 마커/필터링 기능, 한국 지도 완벽 지원

Stripe

국내 카드 지원, 개발자 친화적 API, 테스트 환경 완벽

✅ Provider 상태관리 선택 이유
BLoC는 너무 복잡하고, Riverpod는 새로움. Provider는 학습곡선이 낮으면서도 충분한 기능을 제공해 1명 개발자에게 최적.

8️⃣ Layered + MVVM을 선택한 이유

🎯 왜 이 조합인가?

CafeHub는 7주 안에 1명이 완성해야 합니다. Clean Architecture는 강력하지만 학습 시간이 길고, MVVM만으로는 데이터 흐름이 모호합니다. Layered (명확한 계층 분리) + MVVM (상태 관리)의 조합이 가장 현실적입니다.

비교항목 Clean Arch Layered ⭐ Layered + MVVM
학습곡선 높음 (5-7시간) 낮음 (2시간) 낮음 (2시간)
구조 명확도 매우 높음 높음 높음 ✓
테스트 용이성 매우 높음 높음 높음 ✓
7주 완성 가능 어려움 가능 가능 ✓

✅ 장점

  • 계층이 명확해서 책임 구분 쉬움
  • UI와 비즈니스 로직 완전 분리
  • 테스트 작성이 직관적
  • 새 팀원 온보딩 빠름
  • 나중에 Clean Architecture로 마이그레이션 가능

⚠️ 제약

  • 보일러플레이트 코드 많음
  • 파일 수가 증가
  • 초기 설정에 시간 소요
  • 팀원이 패턴을 이해해야 함
